# Tabs

## Install

Integrate with your build to [auto-import](?path=/docs/getting-started--page#integrating), or add an import in your component:

```
import {ATabGroup, ATab, ATabHeading} from "@cisco-ats/atomic-react/ATabs";
// or if you're able to treekshake node_modules
import {ATabGroup, ATab, ATabHeading} from "@cisco-ats/atomic-react";
```

## Usage

### Uncontrolled Mode

<Playground
  code={`<ATabGroup>
  <ATab>One</ATab>
  <ATab>Two</ATab>
  <ATab selected>Three</ATab>
</ATabGroup>
`}
/>

### Controlled Mode

<Playground
  code={`() => {
  const tabs = ["One", "Two", "Three"];
  const [activeTab, setActiveTab] = useState(tabs[2]);
  return (
    <ATabGroup>
      {tabs.map((x) => (
        <ATab
          key={x}
          tabKey={x}
          onClick={() => setActiveTab(x)}
          selected={x === activeTab}>
          {x}
        </ATab>
      ))}
    </ATabGroup>
  );
}
`}
/>

## Links

Tabs will render in the DOM as anchor tags if the `href` property is defined.

<Playground
  code={`<ATabGroup tall>

  <ATab href="https://www.cisco.com" target="_blank">
    Cisco.com
  </ATab>
</ATabGroup>

/>

## Router Integrations

On `ATab`, set the `component` property to your router's link type, for example:

```
// With react-router:
<ATab component={RouterLink} to="/">Tab 1</ATab>
```

## Accessibility Notes

By default, `ATabGroup` components are assigned the [WAI-ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) role of [tablist](https://www.w3.org/TR/wai-aria/#tablist).

By default, `ATab` components are assigned the [WAI-ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) role of [tab](https://www.w3.org/TR/wai-aria/#tab).

The `ATab` `click` event handler is triggered both by the `click` event and by the `keyDown` event for the `Enter` key.

Each `ATab` component is tab-focusable with a visual overlay.
