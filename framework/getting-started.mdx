# Getting Started

## Install

The package is available publicly on GitHub Packages. `npm login` with the scope `@threatgrid`. ([Read more here.](https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-npm-registry))

`npm i @threatgrid/atomic-react`

<br />

### Content Security Policy

If your website has a content security policy, in order for the fonts to load an entry will need to be added:

`font-src https://brand-assets.security.cisco.com`

<br />

## Sass Required

The components import sass modules. The dart-sass (`sass` package) must be installed and integrated into your bundler.

<br />

### Optional Babel Preset and ESLint Config

The Babel preset **automatically imports** components. Add the following to your Babel configuration:

```diff
{
  "presets": [
    "@babel/preset-env",
    "@babel/preset-react",
+    "@threatgrid/atomic-react/babel-preset"
  ]
}
```

Without import statements, ESLint will complain about your source code. The atomic-react ESLint plugin will take care of that. First install the ESLint plugin:

`npm i -D @threatgrid/eslint-config-atomic-react`

Then add the following to your ESLint configuration:

```diff
{
  "extends": [
    "eslint:recommended",
    "plugin:react/recommended",
+    "@threatgrid/eslint-config-atomic-react"
  ],
  ...
}
```

To use the babel preset and eslint config with `create-react-app`, use `craco`.

For usage with `next`, use `next-transpile-modules`
